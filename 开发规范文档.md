# 开发规范文档

项目名称：ModelGate  
版本：v1.0  
适用范围：前端、后端、Worker、配置、文档、测试

---

## 1. 开发原则

- 安全边界优先。
- 后端是唯一安全控制点。
- Provider 差异隔离在 Adapter。
- 模型能力来自 Model Registry。
- PostgreSQL 是最终事实来源。
- Redis 不保存最终事实。
- 文件路径不暴露给前端。
- API Key 不进入前端和日志。

---

## 3. 前端规范

技术栈：

- Next.js。
- TypeScript。
- Tailwind CSS。
- shadcn/ui。
- Zustand。
- TanStack Query。
- React Hook Form。
- Zod。

规则：

- 组件用 PascalCase。
- hooks 用 `useXxx`。
- store 用 `useXxxStore`。
- API client 统一放在 `src/lib/api`。
- 服务端数据交给 TanStack Query。
- 本地 UI 状态交给 Zustand。
- 表单校验使用 Zod。
- 不在组件中硬编码模型能力。
- 不在前端保存 API Key。

---

## 4. 后端规范

技术栈：

- FastAPI。
- Pydantic。
- SQLAlchemy 2.x。
- Alembic。
- httpx.AsyncClient。
- Celery。

分层：

```text
api -> service -> repository -> db
api -> runtime -> adapter -> provider
worker -> service -> repository
```

规则：

- API 层只做请求校验、鉴权、调用 service。
- Service 层处理业务编排。
- Repository 层处理数据库访问。
- Adapter 层处理 Provider 差异。
- Worker 不绕过状态机。
- 不能在 `async def` 中使用阻塞 HTTP client。

---

## 5. 命名规范

Python：

- 文件：snake_case。
- 函数：snake_case。
- 类：PascalCase。
- 常量：UPPER_SNAKE_CASE。

TypeScript：

- 组件：PascalCase。
- 函数：camelCase。
- 类型：PascalCase。
- 文件：按项目约定，组件 PascalCase，工具 camelCase。

数据库：

- 表名：复数 snake_case。
- 字段名：snake_case。
- JSON 字段后缀：`_json`。

---

## 6. 配置规范

配置文件：

```text
configs/providers.json
configs/models.json
configs/capabilities.json
configs/task-types.json
configs/param-schemas.json
```

规则：

- 新增模型必须通过配置校验。
- 新增 Provider 必须有 Adapter。
- 新增 paramsSchema 必须有后端校验。
- 火山 Seedance 第一版必须 disabled。
- Provider API Key 只通过 envKey 读取。

---

## 7. 数据库规范

- 所有 schema 变更必须通过 Alembic。
- 不允许手动改生产数据库结构。
- 核心查询字段必须独立建列。
- JSONB 只存动态结构。
- 删除字段前必须考虑历史数据兼容。
- migration 文件必须进入版本管理。

---

## 8. API 规范

- 所有 API 使用 `/api` 前缀。
- 所有错误使用标准错误响应。
- 所有响应包含 requestId 或可通过日志追踪。
- 写接口预留 `Idempotency-Key`。
- 文件接口不返回物理路径。
- Provider 原始响应不直接返回前端。

---

## 9. 日志规范

- 使用结构化日志。
- 每条关键日志包含 requestId。
- 日志必须脱敏。
- request_logs 保存脱敏请求和响应。
- 不记录完整 Authorization、api-key、token、secret。

---

## 10. Git 规范

建议分支：

```text
main
dev
feature/*
fix/*
docs/*
```

提交信息建议：

```text
docs: add provider adapter spec
feat: add model registry loader
fix: sanitize provider logs
test: add capability router tests
```

禁止提交：

- `.env`
- API Key。
- `storage/uploads`。
- `storage/outputs`。
- 大型生成文件。
- 本地缓存。

---

## 11. OpenAPI 与类型生成

FastAPI 自动生成：

```text
/openapi.json
```

后续可生成前端类型：

```text
openapi-typescript http://localhost:8000/openapi.json -o apps/web/src/types/api.ts
```

规则：

- API schema 变更后同步更新前端类型。
- 前端不手写和后端重复的 API 类型。
- CI 后续可检查类型是否过期。

---

## 12. 禁止事项

- 禁止前端直连 Provider。
- 禁止前端保存 API Key。
- 禁止上传文件被执行。
- 禁止 Provider 原始错误直接返回前端。
- 禁止 Redis 作为最终任务状态。
- 禁止 Adapter 直接操作业务表。
- 禁止 Worker 绕过状态机。
- 禁止模型输出修改系统配置。
- 禁止在业务代码中硬编码模型能力。

---

## 13. 验收标准

- 新增 Provider 时只需新增配置、Adapter 和测试。
- 新增模型时优先只改 Model Registry。
- 数据库变更都有 Alembic migration。
- 前端无 API Key。
- 日志无敏感字段。
- 代码分层清晰，API、Service、Adapter、Repository 不互相越界。
