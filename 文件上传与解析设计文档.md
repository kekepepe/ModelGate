# 文件上传与解析设计文档

项目名称：ModelGate  
版本：v1.0  
适用范围：文件上传、文件安全、文件解析、预览、模型输入准备

---

## 1. 目标

文件系统要支持多模态任务，同时保持安全边界清晰。

目标：

- 支持图片、视频、音频、文档和代码文件。
- 文件上传不造成路径穿越、XSS 或资源耗尽。
- 文件解析结果可参与模型推荐和任务执行。
- 前端不接触服务器物理路径。
- Adapter 不处理文件安全问题。

---

## 2. 支持文件类型

| 类型 | 扩展名 | detectedType |
|---|---|---|
| 图片 | jpg, jpeg, png, webp, gif | image |
| 视频 | mp4, mov, webm | video |
| 音频 | mp3, wav, m4a | audio |
| 文档 | pdf, docx, txt, md, csv, xlsx, pptx | document |
| 代码 | py, js, ts, tsx, html, css, java, cpp, go, rs, json, yaml | code |

第一版不支持：

- 可执行文件。
- 压缩包自动解压。
- Office 宏执行。
- 上传插件并执行。
- HTML / SVG inline 执行。

---

## 3. 上传流程

```text
前端选择文件
↓
POST /api/files/upload
↓
后端生成 fileId
↓
校验大小、扩展名、MIME、文件头
↓
保存到 storage/uploads
↓
写入 files: uploaded
↓
需要解析则投递 Worker
↓
Worker 解析并写回 metadata
↓
files: parsed 或 failed
```

---

## 4. 文件命名与路径

禁止：

- 使用用户原始文件名作为真实路径。
- 使用用户输入拼接路径。
- 返回服务器物理路径。

存储命名：

```text
storage/uploads/{yyyy}/{mm}/{fileId}.{safeExt}
storage/previews/{yyyy}/{mm}/{fileId}.png
storage/outputs/{yyyy}/{mm}/{outputId}.{ext}
```

数据库：

```text
original_name: 用户原始文件名
stored_path: 内部路径或对象存储 key
preview_path: 预览路径或 key
```

API 只返回：

```text
/api/files/{fileId}/preview
/api/files/{fileId}
```

## 5. 安全校验

### 5.1 必须校验

- 扩展名。
- MIME。
- 文件头 magic number。
- 单文件大小。
- 总上传大小。
- 并发上传数。
- 文件名长度。

### 5.2 建议默认限制

| 类型 | 单文件上限 |
|---|---|
| image | 20 MB |
| video | 500 MB |
| audio | 100 MB |
| document | 100 MB |
| code | 5 MB |

这些值属于第一版建议，可在开发前决策中确认。

### 5.3 响应头

预览与下载接口建议：

```http
X-Content-Type-Options: nosniff
Content-Disposition: inline; filename="safe-name"
```

对 HTML / SVG：

```http
Content-Disposition: attachment
```

## 6. 文件状态

```text
uploading
uploaded
parsing
parsed
failed
deleted
```

规则：

- `uploaded` 表示文件保存成功。
- `parsing` 表示 Worker 正在解析。
- `parsed` 表示解析完成，可进入模型推荐。
- `failed` 表示解析失败，但可根据任务决定是否仍可使用原文件。
- `deleted` 表示逻辑删除。

进入模型推荐条件：

```text
status = parsed
OR
status = uploaded AND direct_usable = true
```

## 7. 图片处理

提取 metadata：

- width。
- height。
- format。
- color mode。
- has alpha。
- file size。

生成：

- preview png。
- thumbnail。

作为 Provider 输入：

- 支持 URL 的 Provider：使用 signed URL 或可访问对象存储 URL。
- 支持 base64 的 Provider：生成 data URL。
- 支持素材库的 Provider：后续可上传为 provider asset ID。

安全：

- SVG 不 inline 渲染。
- GIF 只作为图片输入，不执行脚本。

---

## 8. 视频处理

提取 metadata：

- duration。
- width。
- height。
- fps。
- codec。
- audio codec。
- container。
- file size。

生成：

- poster frame。
- thumbnails。
- 抽帧结果，带时间戳。

抽帧策略：

```text
短视频：均匀抽 5-10 帧
长视频：按场景或固定间隔抽帧
```

第一版可先做固定间隔抽帧。

Provider 输入：

- 火山 Seedance 2.0 后续接入时支持视频 URL 或素材 ID；第一版暂不接入 Seedance。
- 如果 Provider 不支持直接视频输入，则抽帧后走 vision_understanding。

---

## 9. 音频处理

提取 metadata：

- duration。
- format。
- sample rate。
- channels。
- bitrate。

第一版策略：

- 保存原文件。
- 可为后续 audio_transcription 预留。
- 如果 Provider 支持音频输入，可转换为 URL 或 base64 data URL。

火山 Seedance 2.0 后续接入时可接受音频作为部分多模态参考，但不能单独只输入音频生成视频；第一版暂不接入 Seedance。

## 10. 文档处理

### 10.1 PDF

解析：

- 文本。
- 页码。
- 页面截图，按需。
- 表格，按能力提取。

metadata：

- page_count。
- title。
- author。
- encrypted。

### 10.2 DOCX

解析：

- 段落。
- 标题层级。
- 表格。
- 图片引用信息。

不执行：

- 宏。
- 外部链接。

### 10.3 PPTX

解析：

- slide number。
- 文本框。
- 备注。
- 图片 metadata。

### 10.4 XLSX / CSV

解析：

- sheet 名。
- 行列数量。
- 表头。
- 前 N 行样本。
- 可转换为 Markdown 表格或 JSON。

### 10.5 TXT / MD

解析：

- 纯文本。
- Markdown 标题。
- 代码块。

## 11. 代码文件处理

提取：

- language。
- line count。
- file extension。
- text content。

限制：

- 不执行代码。
- 不安装依赖。
- 不解析远程 import。

上下文组织：

```text
BEGIN_CODE_FILE path="src/app.ts" language="typescript"
...
END_CODE_FILE
```

---

## 12. 文件内容进入 Prompt 的边界

文件内容必须作为用户上下文：

```text
BEGIN_USER_FILE_CONTEXT fileId="file_123"
...
END_USER_FILE_CONTEXT
```

禁止：

- 拼入 system prompt。
- 让文件内容覆盖系统指令。
- 让文件内容触发本地命令。

---

## 13. Worker 解析任务

任务类型：

```text
parse_image
parse_video
parse_audio
parse_document
parse_code
generate_preview
```

Worker 输出：

```json
{
  "metadata": {},
  "previewPath": "storage/previews/2026/05/file_123.png",
  "chunks": [],
  "error": null
}
```

解析失败：

- files.status = failed。
- error_message 写入脱敏错误。
- 前端展示失败原因。

## 14. StorageService

接口：

```python
class StorageService:
    async def save_upload(file) -> StoredFile: ...
    async def get_file_stream(file_id: str) -> AsyncIterator[bytes]: ...
    async def get_preview_stream(file_id: str) -> AsyncIterator[bytes]: ...
    async def save_output(bytes_data: bytes, ext: str) -> StoredOutput: ...
    async def create_signed_url(storage_key: str) -> str: ...
```

本地实现：

```text
LocalStorageService
```

生产实现：

```text
ObjectStorageService
```

业务代码只依赖 StorageService，不硬编码本地路径。

---

## 15. 验收标准

- 上传文件名不能造成路径穿越。
- SVG / HTML 不会被 inline 执行。
- 超大文件会被拒绝。
- 文件预览接口不返回物理路径。
- 文件 metadata 能参与模型推荐。
- 删除文件后，前端表单中的 fileIds 同步清理。
- 解析失败不会阻塞其他文件。
- Redis 清空不影响 files 表中已解析 metadata。
