# 前端状态管理与页面交互规范

项目名称：ModelGate  
版本：v1.0  
前端技术栈：Next.js + TypeScript + Tailwind CSS + shadcn/ui + Zustand + TanStack Query + React Hook Form + Zod

---

## 1. 目标

前端状态管理的目标是保持工作台交互稳定，避免任务切换、模型切换、文件删除、参数表单和运行状态互相污染。

原则：

- 服务端数据交给 TanStack Query。
- 本地 UI 状态交给 Zustand。
- 表单状态交给 React Hook Form。
- 参数校验交给 Zod。
- API Key 不进入前端状态。

---

## 2. 状态分工

| 类型 | 工具 | 示例 |
|---|---|---|
| 服务端数据 | TanStack Query | providers、models、files、runs、logs |
| 本地 UI 状态 | Zustand | selectedTaskType、selectedModelId、layout |
| 表单状态 | React Hook Form | prompt、params |
| 表单校验 | Zod | paramsSchema 校验 |
| URL 状态 | Next.js router | taskType、runId、taskId |

## 3. Zustand Store 拆分

```text
stores/
├── workspaceStore.ts
├── fileUiStore.ts
├── modelSelectionStore.ts
└── generationUiStore.ts
```

### 3.1 workspaceStore

保存：

- selectedTaskType。
- activePanel。
- currentRunId。
- currentGenerationTaskId。
- layout。

不保存：

- run 详情。
- generation task 详情。
- Provider 响应。

### 3.2 modelSelectionStore

保存：

- selectedProviderId。
- selectedModelId。
- providerFilter。
- capabilityFilter。

不保存：

- 完整模型列表。
- 推荐结果。

模型列表和推荐结果由 TanStack Query 管理。

### 3.3 fileUiStore

保存：

- 上传队列 UI 状态。
- 拖拽状态。
- 当前选中文件 ID。

不保存：

- 文件真实 metadata。
- 文件内容。
- 文件物理路径。

### 3.4 generationUiStore

第一版 Seedance 暂不接入，但保留 UI 状态：

- 当前 generation task tab。
- 轮询开关。
- 结果面板展开状态。

不保存最终任务状态。

## 4. TanStack Query 管理范围

Query keys：

```ts
["providers"]
["models", filters]
["model", modelId]
["recommendModels", taskType, fileIds, requiredOutput]
["paramSchema", schemaId]
["file", fileId]
["run", runId]
["generationTask", taskId]
["history", filters]
["requestLogs", filters]
```

规则：

- 服务端返回的数据不长期复制到 Zustand。
- mutation 成功后 invalidate 相关 query。
- 文件上传成功后 invalidate `["file", fileId]` 和推荐模型 query。
- 模型切换后重新请求 paramsSchema。

## 5. 页面交互流程

### 5.1 选择任务

```text
用户点击 TaskCard
↓
setSelectedTaskType(taskType)
↓
清空 selectedModelId
↓
清空参数表单
↓
保留已上传文件
↓
触发 recommendModels query
```

如果已上传文件与新任务不兼容：

- 显示提示。
- 不自动删除文件。
- 由用户手动移除或换任务。

### 5.2 上传文件

```text
选择文件
↓
fileUiStore 添加 uploading 状态
↓
POST /api/files/upload
↓
成功后进入 uploaded/parsing
↓
TanStack Query 轮询 file 状态
↓
parsed 后触发模型推荐
```

删除文件：

```text
DELETE /api/files/{fileId}
↓
从表单 fileIds 中移除
↓
invalidate recommendModels
```

### 5.3 选择模型

```text
setSelectedModelId(modelId)
↓
GET /api/param-schemas/{schemaId}
↓
用 default 重置表单
↓
清空旧模型参数
```

禁止：

- 旧模型参数残留到新模型。
- 前端提交 schema 外字段。

### 5.4 运行 Chat

```text
提交 prompt + params + fileIds
↓
POST /api/chat/runs
↓
创建 run
↓
保存 currentRunId
↓
查询 run 详情
↓
展示结果
```

第一版流式输出非硬性要求：

- 如果 `ENABLE_STREAMING=false`，使用普通 HTTP 响应。
- 如果开启 SSE，再接入 `/events`。

## 6. 动态参数表单

规则：

- 表单字段由 paramsSchema 渲染。
- 默认值来自 schema。
- Zod schema 由 paramsSchema 生成。
- 后端仍做 Pydantic 校验。

提交前处理：

- 删除 schema 外字段。
- 保留 `false`。
- 保留 `0`。
- 空字符串按字段类型处理。

---

## 7. 错误展示

错误位置：

| 错误类型 | 展示位置 |
|---|---|
| VALIDATION_ERROR | 表单字段 |
| MODEL_NOT_COMPATIBLE | 模型选择区 |
| FILE_TOO_LARGE | 文件上传区 |
| FILE_PARSE_ERROR | 文件列表 |
| PROVIDER_AUTH_ERROR | Provider 状态 / toast |
| PROVIDER_RATE_LIMIT | toast |
| PROVIDER_TIMEOUT | 结果区重试提示 |

所有错误展示 requestId。

---

## 8. 页面状态恢复

URL 可携带：

```text
/workspace?taskType=chat&runId=run_123
```

恢复流程：

- 从 URL 读取 taskType。
- 加载 run 详情。
- 恢复 selectedModelId。
- 恢复只读结果，不恢复旧表单草稿。

---

## 9. 安全边界

前端禁止保存：

- API Key。
- Authorization。
- Provider 原始响应。
- 文件物理路径。
- 数据库 ID 以外的敏感凭据。

前端允许保存：

- modelId。
- providerId。
- taskType。
- fileId。
- runId。
- generationTaskId。

---

## 10. 验收标准

- 切换任务会清空模型和参数。
- 切换模型会重置参数表单。
- 删除文件会清理 fileIds。
- 服务端数据不重复长期存入 Zustand。
- 错误展示包含 requestId。
- 前端状态中不存在 API Key。
- 第一版非流式 Chat 可以完整运行。
