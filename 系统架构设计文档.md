# 系统架构设计文档

项目名称：ModelGate  
版本：v1.0  
技术栈：Next.js + TypeScript + Tailwind CSS + shadcn/ui + Zustand / FastAPI / PostgreSQL / Redis  
相关文档：`产品需求文档.md`、`技术规格文档.md`、`安全边界与App设计边界.md`、`技术风险与问题清单.md`

---

## 1. 架构目标

ModelGate 的架构目标是把多 Provider、多模型、多任务、多模态文件和异步生成任务统一到一个可扩展、可校验、可追踪的工作台中。

核心架构原则：

- 前端只负责交互体验，不负责安全边界。
- 后端是唯一安全控制点。
- Provider 差异必须隔离在 Adapter 层。
- 模型能力必须来自 Model Registry，不写死在 UI。
- Chat 类任务和 Generation 类任务分离。
- PostgreSQL 是最终事实来源。
- Redis 只用于缓存、队列、锁和短期状态。
- 所有任务、文件、请求、错误和结果必须可追踪。

## 3. 模块边界

| 模块 | 职责 | 禁止事项 |
|---|---|---|
| Frontend | UI、任务选择、上传体验、参数表单、结果展示 | 保存 API Key、直连 Provider、做最终权限判断 |
| FastAPI API | 鉴权、请求校验、调用 Service、标准响应 | 长时间轮询、阻塞式文件处理、拼 Provider 请求 |
| Model Registry | Provider、Model、Capability、TaskType、ParamSchema 配置事实来源 | 根据模型名猜能力 |
| Capability Router | 模型推荐、兼容性校验、hidden reason | 直接调用 Provider |
| Chat Runtime | 文本、代码、视觉理解、文档分析的统一编排 | 处理视频生成异步轮询 |
| Generation Runtime | 生图、生视频任务创建和状态编排 | 同步等待 Provider 生成完成 |
| Provider Adapter | 鉴权、参数映射、响应解析、错误归一化 | 决定业务权限、暴露原始错误 |
| Worker | 文件解析、视频抽帧、生成任务轮询、结果下载 | 绕过状态机写任务 |
| PostgreSQL | 最终数据与历史记录 | 作为临时队列 |
| Redis | 队列、缓存、锁、短期状态 | 作为最终任务状态 |
| StorageService | 文件保存、预览、下载、逻辑 URL | 暴露物理路径 |

### 4.3 状态管理

Zustand 只保存本地 UI 状态：

- selectedTaskType。
- selectedModelId。
- 当前表单草稿。
- 当前上传队列 UI 状态。
- 工作台布局状态。

TanStack Query 管理服务端数据：

- Provider 列表。
- 模型列表。
- 推荐模型结果。
- 文件详情。
- run 详情。
- generation task 状态。
- 历史记录。
- request logs。

## 6. Provider 接入架构

### 6.1 Provider 类型

| Provider | 接口类型 | Runtime | 备注 |
|---|---|---|---|
| Xiaomi MiMo | OpenAI-compatible / Anthropic-compatible | Chat Runtime | 第一接入；模型：MiMo-V2.5-Pro、MiMo-V2.5 |
| MiniMax | Anthropic-compatible / OpenAI-compatible | Chat Runtime | 第二接入；模型：MiniMax-M2.7-highspeed、MiniMax-M2.7 |
| 火山 Coding Plan | OpenAI-compatible / Anthropic-compatible | Chat Runtime | 第三接入；必须使用 Coding Plan 专用 Base URL |
| 火山 Seedance 视频生成 | Ark video generation async API | Generation Runtime | 第一版暂不接入，仅保留架构预留 |

### 6.2 火山 Coding Plan 边界

火山 Coding Plan 必须使用专用 Base URL：

- Anthropic 协议：`https://ark.cn-beijing.volces.com/api/coding`
- OpenAI 协议：`https://ark.cn-beijing.volces.com/api/coding/v3`

禁止在 Coding Plan 额度场景中误用：

- `https://ark.cn-beijing.volces.com/api/v3`

原因：该地址不消耗 Coding Plan 额度，可能产生额外费用。

### 6.3 火山视频生成边界

火山视频生成创建任务接口：

```text
POST https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks
```

特点：

- 第一版暂不接入火山 Seedance。
- 仅支持 API Key 鉴权。
- 创建任务后返回任务 ID。
- 需要通过查询任务 API 获取状态和 `video_url`。
- Provider 状态包括 `queued`、`running`、`succeeded`、`failed`、`expired`。
- 本系统必须映射成本地标准状态。

### 6.4 Xiaomi MiMo 边界

MiMo 支持两类兼容接口：

- OpenAI：`https://api.xiaomimimo.com/v1/chat/completions`
- Anthropic：`https://api.xiaomimimo.com/anthropic/v1/messages`

认证方式支持：

- `api-key: $MIMO_API_KEY`
- `Authorization: Bearer $MIMO_API_KEY`

ModelGate 统一采用后端配置的认证方式，前端不感知。

## 7. 核心请求流

### 7.1 模型推荐流

```text
用户选择 taskType / 上传文件
↓
前端请求 POST /api/models/recommend
↓
后端读取文件 metadata
↓
Capability Router 过滤 models
↓
返回 availableModels + hiddenModels
↓
前端展示模型列表和隐藏原因
```

### 7.2 Chat Run 流

```text
用户输入 prompt / 选择文件 / 选择模型
↓
POST /api/chat/runs
↓
Pydantic 校验
↓
Capability Router 二次校验
↓
File Service 准备上下文
↓
Chat Runtime 调用 Provider Adapter
↓
保存 runs / request_logs / usage_logs
↓
返回文本或 SSE 流式事件
```

### 7.3 Generation Task 流

```text
用户创建生图或生视频任务
↓
POST /api/generation/tasks
↓
创建本地 generation_task: queued
↓
Generation Runtime 提交 Provider 任务
↓
保存 providerTaskId: submitted
↓
Worker 轮询 Provider 状态
↓
状态写回 PostgreSQL
↓
成功后下载或记录输出文件
↓
前端轮询 GET /api/generation/tasks/{taskId}
```

## 8. 部署拓扑

### 8.1 本地开发

```text
web: Next.js dev server
api: FastAPI uvicorn
worker: Celery worker
db: PostgreSQL container
redis: Redis container
storage: local folder
```

### 8.2 生产建议

```text
web: Vercel / Node server / container
api: FastAPI container
worker: independent Celery worker container
db: managed PostgreSQL
redis: managed Redis
storage: S3 / R2 / OSS
```

生产环境必须具备：

- API 和 Worker 独立扩缩容。
- PostgreSQL 自动备份。
- Redis 持久化或托管服务。
- 对象存储替代本地磁盘。
- Provider API Key 由 Secret Manager 管理。

## 9. 架构验收标准

- 前端不能直接调用 Provider。
- 前端构建产物中不存在 Provider API Key。
- 模型推荐只来自 Capability Router。
- Chat Runtime 和 Generation Runtime 分离。
- Worker 不绕过状态机写任务状态。
- Redis 重启后，PostgreSQL 仍可查询任务最终状态。
- Provider 原始错误不会直接返回前端。
- 文件下载和预览不暴露物理路径。
