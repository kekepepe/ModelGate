## 低优先级

- **V2.7 Controlled Auto（Celery + sandbox 升级版）**：当前已用 asyncio + 本地 git apply 实现 V2.7 MVP；后续若需要更高隔离 / 多用户并发 / 跨进程 task queue，可单独排期做 Celery + patch sandbox 改造。

- **V2 P2：API Keys 多 key 轮转 + 用量配额**：当前一 provider 一 key，方案建议支持多 key 池 + 每 key 用量上限；需要后端 schema 改造。**已确认推迟到 V3。**
- **Playwright 集成测试套件**：当前只有 smoke spec；如果想做真后端 + 真数据库的 E2E，需要单独 `e2e/integration/` + CI 矩阵。
- **Project Mode Memory 跨 run 复用**：`project_memory` 表已建好，MVP 只在 run 内读写；V2.7 引入跨 run RAG。
- **Worker / Planner 任务取消粒度**：当前 cancel 只整 run 停；未来要支持单 worker 取消 + 重做。
- **project_runs / project_tasks 外键加 `ondelete="CASCADE"`**：现在 `delete_project_run` 用 ORM ordered delete 兜住，schema 层 alembic 0004 的 FK 无 `ondelete`；未来若做大版本 schema 改造，新写 migration 给 5 张表所有引用 `project_runs.id`/`project_tasks.id`/`agent_runs.id` 的 FK 加 CASCADE，删除端点可简化为 `db.delete(pr)` 一句。alter table 对已部署 DB 有风险，单独排期做。
