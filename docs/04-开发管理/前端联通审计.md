# 前端按钮与后端联通审计

更新时间：2026-06-03

本文档是 ModelGate 前端控件 / 页面入口与后端 API 联通状态的审计快照。
原内容位于 `待开发文档.md` 第 1 节，为避免与待开发项混淆，整体迁移到本文件。

审计关系：

- 已联通明细：见第 1 节，作为前端 ↔ 后端联通的当前事实记录。
- 旧组件状态：见第 2 节，记录尚未清理的旧版工作台组件。
- 未联通 / 占位 / 后续接入项：见 `待开发文档.md`，按 P0 / P1 / P2 优先级组织。

---

## 1. 审计范围

- 当前实际入口：`apps/web/src/components/workspace/workspace-shell.tsx`
- 其他现有页面：`history-client.tsx`、`model-management.tsx`、`settings-client.tsx`
- 后端路由：`apps/server/app/api/*`

### 1.1 已联通后端的核心交互

| 前端交互 | 当前行为 | 后端接口 | 状态 |
|---|---|---|---|
| 工作台初始化 Provider | 页面加载读取 Provider | `GET /api/providers` | 已联通 |
| 工作台初始化模型 | 页面加载读取模型列表 | `GET /api/models` | 已联通 |
| 任务类型切换 | 更新 `taskType` 与 URL，并触发模型推荐 query key 变化 | `POST /api/models/recommend` | 已联通 |
| Provider 全部 / 单 Provider 筛选 | 更新 `providerFilter`，触发模型推荐 | `POST /api/models/recommend` | 已联通 |
| 选择模型卡片 | 更新 `selectedModelId`，读取对应参数 Schema | `GET /api/param-schemas/{schemaId}` | 已联通 |
| 动态参数输入 / slider / checkbox / select | 更新本地参数，并参与推荐与运行请求 | `POST /api/models/recommend`、`POST /api/chat/runs` | 已联通 |
| 上传文件按钮 | 打开文件选择器，选择文件后上传 | `POST /api/files/upload` | 已联通 |
| 清除文件按钮 | 删除当前文件记录 | `DELETE /api/files/{fileId}` | 已联通 |
| 运行按钮 | 创建真实 Chat Run | `POST /api/chat/runs` | 已联通 |
| 输出预览 | 展示最新 run 输出或错误 requestId | 来自 `POST /api/chat/runs` 响应 | 已联通 |
| 结果归档 tab | 展示工作台已加载历史 | `GET /api/history/runs` | 已联通 |
| 历史记录页 | 读取运行历史与请求日志 | `GET /api/history/runs`、`GET /api/logs/requests` | 已联通 |
| 模型管理页 | 读取 Provider 与模型 | `GET /api/providers`、`GET /api/models` | 已联通 |
| 设置页 Provider Key 状态 | 只读展示 Provider env key 和启用状态 | `GET /api/providers` | 已联通读能力 |
| 设置页保存 Provider Key | 写入本地 UI key，AES-256-GCM 加密存储，不回显明文 | `PUT /api/providers/{providerId}/key` | 已联通 |
| 设置页清除 Provider Key | 清除本地 UI 写入 key | `DELETE /api/providers/{providerId}/key` | 已联通 |
| 左侧导航：任务中心 / Chat Runtime | 跳转工作台 | `/workspace?taskType=chat` | 已联通 |
| 左侧导航：历史记录 / 请求日志 | 跳转历史与日志页 | `/history`，页面读取 `/api/history/runs`、`/api/logs/requests` | 已联通 |
| 左侧导航：Provider 管理 / 设置 | 跳转设置页 | `/settings`，页面读取 / 写入 `/api/providers` | 已联通 |
| 左侧导航：模型注册表 | 跳转模型页 | `/models`，页面读取 `/api/models` | 已联通 |
| 输出区复制 / 下载 | 复用旧组件能力，本地复制和下载文本结果 | 不需要后端 | 已接入 |
| 历史重跑 | 使用历史 run 的输入重新创建 Chat Run | `POST /api/chat/runs` | 已联通 |
| 参数面板重置 | 恢复 `param-schemas` 默认值 | `GET /api/param-schemas/{schemaId}` | 已联通 |
| 图片文件预览 | 使用后端预览 URL 展示缩略图 | `GET /api/files/{fileId}/preview` | 已联通 |

### 1.3 维护规则

- 新增前端控件时，同步在本表登记对应的后端接口。
- 后端接口路径或语义调整时，校验本表是否仍准确。
- 已废弃的旧组件清理完成后，从 1.2 节移除对应条目。
- 任何未联通 / 占位条目不写在本文件，统一进入 `待开发文档.md`。
