### D-2026-06-04-08 — `.gitignore` 追加 3 类 IP / 隐私 pattern

**决策：** 追加以下 pattern 到 `.gitignore`：

```gitignore
# Third-party / internal-sensitive content (kept on disk, not on GitHub)
**/前端设计参考图/
**/tokenplan_api文档/
*-修改建议.md
```

**Why:**
- `**/前端设计参考图/`：5 张 Firecrawl 截图，是第三方设计参考，本项目不拥有版权也不应公开放。匹配 `**/` 前缀既覆盖根目录的、也覆盖任何深度的同名目录（防御未来同样的命名）。
- `**/tokenplan_api文档/`：4 个厂商的 API 文档（minimax / Xiaomi MiMo / Volcengine / 火山视频生成），版权通常归各厂商，公开放可能违反 ToS。同 `**/` 前缀防御。
- `*-修改建议.md`：捕捉"内部评估反馈"类型文件。当前只有 `minimax3-修改建议.md`，里面 P0 列了 API Key 明文等安全风险细节，公开发相当于把项目安全弱点广而告之。`*-修改建议.md` 是个宽 pattern，未来类似文件会自动被 ignore（防御性）。
- **不**用 `*修改建议.md`（去前导 `-`）—— 太宽，会误伤无关的"修改建议"。
- **不**用 `**/*.md` —— 显然太宽，会把整个项目的 md 文档全 ignore 掉。

**注意：** 这 3 个 pattern 之前未 ignore，所以 `minimax3-修改建议.md` 等目前还是 tracked 状态。Stage 5 需要 `git rm --cached` 把它们从索引里拿掉，但保留工作树文件。

### D-2026-06-04-10 — README 默认语言保持英文，中文放 `README.zh-CN.md`

**决策：** `README.md` 保留英文作为默认入口；新增 `README.zh-CN.md` 提供中文版。两个 README 顶部互相加语言切换链接。

**Why:**
- 与当前 `README.md` 现有语言保持一致（原本就是英文），降低改动面积。
- GitHub 对后缀为 `*.zh-CN.md` / `*.zh.md` / `*.中文.md` 的 README 会**自动**在仓库头显示语言切换器；只要文件存在就行，README 内部可再手动加切换链接作为冗余。
- 国际访客首屏直接看到英文，更符合开源协作惯例；中文母语用户点击右上角语言切换器或 README 顶部链接即可切换。
- 两个 README 完全同构（章节顺序、标题、表格、代码块一致），保证任何语言读者拿到的信息密度相同。
- **不**用单一 `README.md` + YAML frontmatter 写 `lang: zh-CN`——那会强制把默认语言变成中文，违背本决策的第一原则。
