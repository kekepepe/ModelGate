# 环境变量与部署文档

项目名称：ModelGate  
版本：v1.0  
第一版范围：本地单用户 GitHub 项目，无登录；本地存储；优先接入 MiMo、MiniMax、火山 Coding Plan。

---

## 1. 目标

本文档定义第一版本地开发、Docker Compose、环境变量、PostgreSQL、Redis、存储目录和启动流程。

目标：

- clone 项目后能按 README 快速启动。
- API Key 只配置在后端 `.env`。
- 前端不持有 Provider API Key。
- 本地存储目录清晰。
- Docker Compose 可一键启动基础依赖。

---

## 2. 环境文件

建议提供：

```text
.env.example
apps/server/.env.example
apps/web/.env.example
```

第一版可以简化为根目录 `.env.example`，后续拆分。

禁止提交：

```text
.env
.env.local
.env.*.local
apps/server/.env
apps/web/.env
```

## 3. 后端环境变量

### 3.1 基础配置

```env
APP_ENV=development
APP_NAME=ModelGate
API_HOST=0.0.0.0
API_PORT=8000
LOG_LEVEL=INFO
LOG_PARTIAL_SECRETS=false
```

### 3.2 数据库

```env
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=modelgate
POSTGRES_USER=modelgate
POSTGRES_PASSWORD=modelgate_password
DATABASE_URL=postgresql+psycopg://modelgate:modelgate_password@localhost:5432/modelgate
```

### 3.3 Redis

```env
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_DB=0
REDIS_URL=redis://localhost:6379/0
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

### 3.4 Provider API Key

```env
MIMO_API_KEY=
MINIMAX_API_KEY=
VOLCENGINE_API_KEY=
```

预留：

```env
MOONSHOT_API_KEY=
ZHIPU_API_KEY=
```

第一版说明：

- MiMo 使用 `MIMO_API_KEY`。
- MiniMax 使用 `MINIMAX_API_KEY`。
- 火山 Coding Plan 使用 `VOLCENGINE_API_KEY`。
- 火山 Seedance 第一版暂不接入，但后续可复用 `VOLCENGINE_API_KEY` 或单独配置。

### 3.5 存储

```env
STORAGE_DRIVER=local
STORAGE_ROOT=./storage
UPLOADS_DIR=./storage/uploads
OUTPUTS_DIR=./storage/outputs
PREVIEWS_DIR=./storage/previews
```

### 3.6 文件大小限制

```env
MAX_IMAGE_MB=20
MAX_VIDEO_MB=500
MAX_AUDIO_MB=100
MAX_DOCUMENT_MB=100
MAX_CODE_MB=5
```

### 3.7 功能开关

```env
ENABLE_STREAMING=false
ENABLE_SEEDANCE=false
ENABLE_AUTH=false
ENABLE_OBJECT_STORAGE=false
```

第一版默认：

- `ENABLE_STREAMING=false`
- `ENABLE_SEEDANCE=false`
- `ENABLE_AUTH=false`
- `ENABLE_OBJECT_STORAGE=false`

## 4. 前端环境变量

前端只允许保存公开配置：

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api
NEXT_PUBLIC_APP_NAME=ModelGate
```

禁止：

```env
NEXT_PUBLIC_MIMO_API_KEY=
NEXT_PUBLIC_MINIMAX_API_KEY=
NEXT_PUBLIC_VOLCENGINE_API_KEY=
```

任何 `NEXT_PUBLIC_*_API_KEY` 都不允许出现。

## 5. Docker Compose

第一版基础服务：

```yaml
services:
  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: modelgate
      POSTGRES_USER: modelgate
      POSTGRES_PASSWORD: modelgate_password
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U modelgate"]
      interval: 5s
      timeout: 5s
      retries: 10

  redis:
    image: redis:7
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 10

volumes:
  postgres_data:
```

API、Worker、Web 可以第一版先本地运行，后续加入 compose。

## 6. 本地启动流程

### 6.1 环境策略

第一版开发环境采用：

- Python 后端：使用独立 conda 环境，环境名建议为 `modelgate`。
- 前端：使用本机 Node.js，不放进 conda。
- PostgreSQL / Redis：使用 Docker Compose 启动。
- 不建议使用 conda `base` 环境，避免污染全局依赖。
- 全 Docker 启动保留用于后续演示、部署或稳定版本回归。

推荐版本：

```text
Python 3.11
Node.js 20 LTS 或更高 LTS
Docker Desktop
PostgreSQL 16
Redis 7
```

### 6.2 创建后端 conda 环境

```bash
conda create -n modelgate python=3.11
conda activate modelgate
pip install -r apps/server/requirements.txt
```

如果本机不使用 conda，也可以使用项目内 venv：

```bash
python -m venv apps/server/.venv
source apps/server/.venv/bin/activate
pip install -r apps/server/requirements.txt
```

两种方式只选一种；不要直接安装到 conda `base`。

### 6.3 启动依赖服务

```bash
docker compose up -d postgres redis
```

### 6.4 后端

```bash
conda activate modelgate
PYTHONPATH=apps/server alembic -c apps/server/alembic.ini upgrade head
PYTHONPATH=apps/server uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 6.5 Worker

```bash
conda activate modelgate
PYTHONPATH=apps/server celery -A app.workers.celery_app worker --loglevel=info
```

### 6.6 前端

```bash
cd apps/web
npm install
npm run dev
```

默认访问：

```text
http://localhost:3000
```

### 6.7 全 Docker 启动

全 Docker 不是第一阶段开发的主推荐方式，但需要保留为后续演示和部署入口：

```bash
docker compose up --build
```

开发阶段优先使用本机前后端进程，便于调试、热更新和单独查看日志。

## 8. 健康检查

API：

```http
GET /health
```

响应：

```json
{
  "status": "ok",
  "database": "ok",
  "redis": "ok"
}
```

Provider Key 检查：

```http
GET /api/providers/health
```

响应不返回 API Key，只返回是否配置：

```json
{
  "data": [
    {
      "providerId": "mimo",
      "configured": true,
      "enabled": true
    }
  ]
}
```

## 9. 部署边界

第一版建议：

- 本地运行或个人自部署。
- 不直接公网开放。
- 不启用多用户登录。
- 不使用对象存储。

后续生产化需要补：

- Auth。
- HTTPS。
- Secret Manager。
- 对象存储。
- 日志平台。
- 数据库备份。
- Redis 托管服务。

---

## 10. 验收标准

- clone 后可以按 `.env.example` 配置启动。
- 前端环境变量不包含 API Key。
- 后端启动时能检查 PostgreSQL 和 Redis。
- `.env` 不会被提交。
- `storage/` 下上传和输出文件不会进入仓库。
- `ENABLE_SEEDANCE=false` 时不会显示 Seedance 可用模型。
