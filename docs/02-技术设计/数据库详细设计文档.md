# 数据库详细设计文档

项目名称：ModelGate  
版本：v1.0  
数据库：PostgreSQL  
迁移工具：Alembic  
缓存与队列：Redis

---

## 1. 设计原则

- PostgreSQL 是最终事实来源。
- Redis 只保存缓存、队列、锁和短期状态。
- 核心查询字段必须独立建列，不能全部塞入 JSONB。
- Provider 原始响应可以进入日志表，但业务输出必须使用标准结构。
- 所有表必须包含 `created_at`，可变配置表必须包含 `updated_at`。
- 所有 schema 变更必须通过 Alembic migration。

---

## 2. 命名规范

- 表名：复数 snake_case，例如 `generation_tasks`。
- 字段名：snake_case，例如 `provider_task_id`。
- 主键：`id TEXT PRIMARY KEY`，由应用生成。
- 外键：`provider_id`、`model_id`、`run_id`。
- JSON 字段：统一后缀 `_json`。
- 时间字段：`TIMESTAMPTZ`。

## 4. providers

用途：保存 Provider 基础配置，不保存明文 API Key。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| id | TEXT PK | 是 | provider ID，例如 `volcengine` |
| name | TEXT | 是 | 展示名 |
| base_url | TEXT | 是 | 默认 base URL |
| auth_type | TEXT | 是 | `bearer`、`api_key_header`、`custom` |
| env_key | TEXT | 否 | 环境变量名，例如 `VOLCENGINE_API_KEY` |
| adapter | TEXT | 是 | Adapter 名称 |
| enabled | BOOLEAN | 是 | 是否启用 |
| metadata_json | JSONB | 否 | 协议、限流、区域等扩展配置 |
| created_at | TIMESTAMPTZ | 是 | 创建时间 |
| updated_at | TIMESTAMPTZ | 是 | 更新时间 |

索引：

```sql
CREATE INDEX idx_providers_enabled ON providers(enabled);
```

| 字段 | 类型 | 必填 | 说明 |

| capabilities | JSONB | 是 | 能力数组 |
| input_types | JSONB | 是 | 输入类型数组 |
| output_types | JSONB | 是 | 输出类型数组 |
| task_types | JSONB | 是 | 支持任务类型数组 |
| context_window | INTEGER | 否 | 上下文长度 |
| params_schema_id | TEXT | 否 | 参数 schema ID |
| is_async | BOOLEAN | 是 | 是否异步 |
| enabled | BOOLEAN | 是 | 是否启用 |

| created_at | TIMESTAMPTZ | 是 | 创建时间 |

```sql

```

## 14. Redis Key 设计

Redis 不保存最终事实，只保存短期状态。

```text
lock:generation_task:{taskId}          # 分布式锁
progress:generation_task:{taskId}      # 短期进度缓存
queue:celery                           # Celery 队列
cache:models:recommend:{hash}          # 推荐结果短缓存
rate_limit:provider:{providerId}       # Provider 限流计数
```

---

## 15. 迁移与验收

- 所有表由 Alembic migration 创建。
- 所有状态字段由应用层校验，后续可升级为数据库 check constraint。
- 所有 JSONB 必须有文档定义。
- 删除字段前必须确认历史数据兼容策略。
- Redis 清空后系统仍能通过 PostgreSQL 恢复任务最终状态。
