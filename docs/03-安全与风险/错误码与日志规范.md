# 错误码与日志规范

项目名称：ModelGate  
版本：v1.0  
适用范围：FastAPI、Provider Adapter、Runtime、Worker、前端错误展示、request_logs  
第一版范围：本地单用户，无登录；优先接入 MiMo、MiniMax、火山 Coding Plan；火山 Seedance 暂不接入。

---

## 1. 目标

本规范用于统一系统错误码、错误响应、requestId、日志字段、日志脱敏和 Provider 错误映射。

目标：

- 前端能稳定展示错误。
- 后端能快速定位问题。
- Provider 原始错误不会直接暴露。
- 日志不泄露 API Key、Authorization、文件物理路径。
- 所有请求都能通过 requestId 串联追踪。

---

## 2. 标准错误响应

所有 API 错误统一返回：

```json
{
  "error": {
    "type": "MODEL_NOT_COMPATIBLE",
    "message": "当前模型不支持 image_to_video",
    "requestId": "req_123",
    "details": {}
  }
}
```

字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| type | 是 | 标准错误码 |
| message | 是 | 可展示给用户的错误信息 |
| requestId | 是 | 全链路追踪 ID |
| details | 否 | 脱敏后的结构化详情 |

禁止返回：

- Python traceback。
- Provider API Key。
- Authorization header。
- 数据库连接串。
- 服务器物理路径。
- 未脱敏 Provider 原始响应。

---

## 3. requestId 规范

### 3.1 来源

优先级：

1. 前端请求头 `X-Request-Id`。
2. 后端 middleware 自动生成。

格式建议：

```text
req_{ulid}
```

### 3.2 传递范围

requestId 必须写入：

- API 响应。
- 标准错误响应。
- request_logs。
- Provider Adapter 调用日志。
- Worker 任务日志。
- generation_tasks / runs 的 metadata_json，可选。

### 3.3 日志格式

每条日志必须包含：

```json
{
  "requestId": "req_123",
  "level": "INFO",
  "event": "provider.request.completed"
}
```

## 4. 错误码列表

### 4.1 通用错误

| 错误码 | HTTP | 是否可重试 | 说明 |
|---|---:|---|---|
| VALIDATION_ERROR | 400 | 否 | 请求体、参数或 schema 校验失败 |
| NOT_FOUND | 404 | 否 | 资源不存在 |
| METHOD_NOT_ALLOWED | 405 | 否 | 方法不允许 |
| CONFLICT | 409 | 否 | 状态冲突或幂等键冲突 |
| INTERNAL_ERROR | 500 | 视情况 | 系统内部错误 |
| UNKNOWN_ERROR | 500 | 视情况 | 未归类错误 |

### 4.2 模型与任务错误

| 错误码 | HTTP | 是否可重试 | 说明 |
|---|---:|---|---|
| MODEL_NOT_COMPATIBLE | 400 | 否 | 模型不支持当前 taskType、inputTypes 或 outputTypes |
| MODEL_NOT_FOUND | 404 | 否 | 模型不存在或未启用 |
| PROVIDER_DISABLED | 400 | 否 | Provider 未启用 |
| PARAM_SCHEMA_NOT_FOUND | 500 | 否 | 模型绑定的 paramsSchema 不存在 |
| TASK_NOT_FOUND | 404 | 否 | run 或 generation_task 不存在 |
| TASK_STATE_CONFLICT | 409 | 否 | 状态机不允许该操作 |
| TASK_EXPIRED | 408 | 否 | 任务已过期 |
| IDEMPOTENCY_CONFLICT | 409 | 否 | 相同幂等键对应不同请求内容 |

### 4.3 文件错误

| 错误码 | HTTP | 是否可重试 | 说明 |
|---|---:|---|---|
| FILE_NOT_FOUND | 404 | 否 | 文件不存在 |
| FILE_TOO_LARGE | 413 | 否 | 文件超过大小限制 |
| FILE_TYPE_NOT_SUPPORTED | 400 | 否 | 文件类型不支持 |
| FILE_PARSE_ERROR | 422 | 否 | 文件解析失败 |
| FILE_NOT_READY | 409 | 是 | 文件仍在上传或解析 |
| STORAGE_ERROR | 500 | 是 | 本地存储或对象存储错误 |

### 4.4 Provider 错误

| 错误码 | HTTP | 是否可重试 | 说明 |
|---|---:|---|---|
| PROVIDER_AUTH_ERROR | 502 | 否 | API Key 错误、无权限或认证失败 |
| PROVIDER_RATE_LIMIT | 429 | 是 | Provider 限流 |
| PROVIDER_TIMEOUT | 504 | 是 | Provider 请求超时 |
| PROVIDER_BAD_REQUEST | 502 | 否 | Provider 拒绝参数 |
| PROVIDER_TASK_FAILED | 502 | 视情况 | Provider 任务失败 |
| PROVIDER_UNAVAILABLE | 503 | 是 | Provider 服务不可用 |
| PROVIDER_RESPONSE_INVALID | 502 | 否 | Provider 响应无法解析 |

## 5. Provider 错误映射

### 5.1 HTTP 状态映射

| Provider HTTP | ModelGate 错误码 |
|---:|---|
| 400 | PROVIDER_BAD_REQUEST |
| 401 | PROVIDER_AUTH_ERROR |
| 403 | PROVIDER_AUTH_ERROR |
| 404 | MODEL_NOT_FOUND 或 PROVIDER_TASK_FAILED |
| 408 | PROVIDER_TIMEOUT |
| 429 | PROVIDER_RATE_LIMIT |
| 500 | PROVIDER_UNAVAILABLE |
| 502 | PROVIDER_UNAVAILABLE |
| 503 | PROVIDER_UNAVAILABLE |
| 504 | PROVIDER_TIMEOUT |

### 5.2 火山视频状态映射

火山 Seedance 第一版暂不接入，但预留状态映射：

| Provider 状态 | ModelGate 状态 | 错误码 |
|---|---|---|
| queued | submitted | 无 |
| running | processing | 无 |
| succeeded | completed | 无 |
| failed | failed | PROVIDER_TASK_FAILED |
| expired | expired | TASK_EXPIRED |

### 5.3 MiMo / MiniMax / 火山 Coding Plan Chat

Chat Provider 错误统一进入：

- 认证失败：`PROVIDER_AUTH_ERROR`
- 限流：`PROVIDER_RATE_LIMIT`
- 超时：`PROVIDER_TIMEOUT`
- 参数错误：`PROVIDER_BAD_REQUEST`
- 响应解析失败：`PROVIDER_RESPONSE_INVALID`

## 6. 日志级别

| 级别 | 使用场景 |
|---|---|
| DEBUG | 本地开发调试，不记录敏感字段 |
| INFO | 请求开始、请求完成、任务状态变化 |
| WARNING | 可恢复异常、Provider 限流、重试 |
| ERROR | 请求失败、任务失败、解析失败 |
| CRITICAL | 服务不可用、数据库连接失败、配置严重错误 |

生产默认：

```text
INFO
```

## 7. 结构化日志字段

通用字段：

```json
{
  "timestamp": "2026-05-27T10:00:00Z",
  "level": "INFO",
  "event": "chat.run.completed",
  "requestId": "req_123",
  "recordType": "run",
  "recordId": "run_123",
  "providerId": "mimo",
  "modelId": "mimo.mimo_v2_5_pro",
  "taskType": "chat",
  "latencyMs": 1200,
  "status": "completed"
}
```

Provider 请求日志：

```json
{
  "event": "provider.request.completed",
  "requestId": "req_123",
  "providerId": "mimo",
  "modelId": "mimo.mimo_v2_5_pro",
  "statusCode": 200,
  "latencyMs": 900,
  "retryCount": 0
}
```

Worker 日志：

```json
{
  "event": "worker.task.completed",
  "taskName": "parse_document",
  "fileId": "file_123",
  "requestId": "req_123",
  "latencyMs": 3200
}
```

## 8. request_logs 表记录

必须保存：

- record_type。
- record_id。
- provider_id。
- model_id。
- request_json，脱敏。
- response_json，脱敏。
- status_code。
- latency_ms。
- error_type。
- error_message。
- created_at。

不保存：

- 完整 API Key。
- 完整 Authorization。
- 数据库密码。
- 服务器物理路径。
- 未脱敏文件内容。

---

## 9. 脱敏规则

### 9.1 必须脱敏字段

字段名包含以下关键词时必须脱敏：

```text
authorization
api-key
apikey
api_key
token
secret
password
credential
cookie
set-cookie
```

脱敏后：

```text
[REDACTED]
```

### 9.2 部分保留

调试 API Key 所属 Provider 时可保留前后几位：

```text
sk-abc...xyz
```

默认不启用，除非明确设置：

```text
LOG_PARTIAL_SECRETS=true
```

第一版建议保持 `false`。

### 9.3 文件路径脱敏

内部路径：

```text
/Users/name/project/storage/uploads/2026/05/file_123.png
```

日志中只能记录：

```text
storage_key: uploads/2026/05/file_123.png
```

## 10. 前端错误展示

前端按错误码决定展示方式：

| 错误码 | 展示方式 |
|---|---|
| VALIDATION_ERROR | 表单字段错误或 toast |
| MODEL_NOT_COMPATIBLE | 模型选择区提示 |
| FILE_TOO_LARGE | 上传区域提示 |
| FILE_PARSE_ERROR | 文件列表错误状态 |
| PROVIDER_AUTH_ERROR | 设置页 / Provider 状态提示 |
| PROVIDER_RATE_LIMIT | toast + 稍后重试 |
| PROVIDER_TIMEOUT | toast + 重试按钮 |
| TASK_STATE_CONFLICT | 当前任务状态提示 |

前端必须展示 requestId，方便定位：

```text
请求 ID：req_123
```

---

## 11. 验收标准

- 所有错误响应都有 `error.type`、`error.message`、`error.requestId`。
- request_logs 中没有完整 API Key。
- Provider 原始错误不会直接返回给前端。
- 前端能根据错误码展示合适位置的错误。
- Worker 日志能通过 requestId 或 taskId 追踪。
- 日志中没有服务器物理路径。
